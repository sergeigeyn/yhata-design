## 1. Что такое Claude Design (суть)

- **Запуск:** 17 апреля 2026, Anthropic Labs

- **Статус:** Research Preview

- **Философия Anthropic:** «Code is the universal medium for all knowledge work» — видео, слайды, дизайн, веб — всё это код, который можно генерировать быстро

## 2. Архитектура объектов (важно понять)

Claude Design состоит из **трёх сущностей** (не «проектов», как может показаться):

| Объект | Что это | Когда создавать |
|---|---|---|
| **Design** | Отдельный прототип / макет | Конкретная задача: «лендинг hero», «экран онбординга» |
| **Design System** | Переиспользуемый набор правил, цветов, шрифтов, компонентов | Для бренда — **один раз**, потом подключается ко всем Designs |
| **Slide deck / Template / Other** | Альтернативные форматы | Презентации, шаблоны |

**Ключевой инсайт:** Design System создаётся **отдельно**, а потом выбирается в dropdown при создании Design. Если создать Design с `Design system: None` — бренд не применится.

## 3. Режимы создания (левая колонка на главном экране)

Четыре типа проектов (консенсус 14 видеообзоров + официальный анонс):

- **Prototype** — прототипы сайтов и приложений. Два режима:
  - `Wireframe` — черновик (скелет без дизайна)
  - `High Fidelity` — готовый внешний вид с цветами/шрифтами/картинками
- **Slide Deck** — презентации. **Фича:** режим `Speaker Notes` → Claude сам пишет примерный текст выступления к каждому слайду
- **From Template** — использование прежних работ как шаблонов (пока почти пустой раздел — только `animation`)
- **Other** — всё остальное (постеры, флаеры, визитки, меню, приглашения)

## 4. Что Claude Design умеет делать

1. **Веб-сайты/лендинги** — за один промпт
2. **Мобильные приложения (прототипы)** — экраны, не работающий код
3. **Презентации** — с автогенерацией speaker notes
4. **Анимации** — работают (3D-глобус, scroll-triggered появление элементов). Из коробки, не через внешние либы
5. **Видео с анимацией** — **но нет экспорта в файл** (только screen recording)
6. **Дизайн-системы** — главная фича, полные системы компонентов (пример: Apple Liquid Glass)
7. **Одноразовые материалы** — постеры, меню, визитки, именные карточки (~10 мин работы)

**КРИТИЧНОЕ ОГРАНИЧЕНИЕ:** Claude Design **НЕ генерирует картинки**. Их нужно либо загружать вручную, либо подключать внешний сервис через Claude Code.

## 4.5 Четыре способа редактирования готового дизайна ⭐

Критично для экономии лимитов:

| Режим | Что делает | Расходует токены? |
|---|---|---|
| **Comment** | Кликаешь на элемент → пишешь текстом правку («сделай кнопку больше») | ✓ Да |
| **Edit** | Панель свойств (цвет, размер, шрифт) — правишь руками | ❌ **НЕТ** |
| **Draw** | Рисуешь поверх стрелки/обводки/пометки | ✓ Да |
| **Tweaks** | Слайдеры стиля («кинематографичный»↔«журнальный», закруглённость, яркость) | ✓ Да |

**ПРАВИЛО ЭКОНОМИИ:** мелкие правки (изменить цвет кнопки, подвинуть элемент) → всегда через **Edit**. Модель не работает, токены не расходуются. Промпты — только для концептуальных изменений.

## 4.6 Handoff в Claude Code — как работает передача

1. Нажимаешь `Handoff to Claude Code` в Claude Design
2. Claude Design упаковывает весь дизайн в архив + копирует команду в буфер обмена
3. Вставляешь команду в окно Claude Code
4. Локальный Claude Code сам скачивает архив, распаковывает и собирает проект на машине

**Безопасность (важно):** код через локальный путь **НЕ загружается** на серверы Anthropic и **НЕ используется** для обучения модели. Хранится только «представление дизайн-системы», не исходники.

## 5. Интерфейс создания Design System (заполняется анкетой)

Три поля:

### A. Company name and blurb
Короткое описание бренда — Claude читает и формирует mental model.

### B. Provide examples (опционально)
- **Link code on GitHub** — подключить репо (Claude читает код)
- **Link code from your computer** — только Chrome/Edge, выбранные файлы
- **Upload a .fig file** — Figma-файл (парсится локально в браузере, не загружается)
- **Add fonts, logos and assets** — **ГЛАВНОЕ поле для нас** — сюда грузим WOFF2 + SVG логотипы

### C. Any other notes?
**Критичное поле.** Сюда идут все бренд-правила текстом — палитра, типографика, компоненты, voice, запреты. Чем подробнее, тем точнее модель.

После `Continue to generation →` Claude сам превращает notes + ассеты в Design System с токенами, компонентами, превью.

## 6. Best Practices (общие)

1. **Сначала Design System, потом Prototype** — иначе прототип будет без бренда
2. **Notes — главное оружие.** Загрузка ассетов без текстового описания даёт слабый результат. Всегда пиши плотный текст правил.
3. **Структурируй notes по секциям** — `=== COLORS ===`, `=== TYPOGRAPHY ===`, `=== HARD RULES ===`. Модель лучше читает маркированные блоки.
4. **Jarvis-правило:** для слайдов — сначала обработай копирайт в обычном Claude, потом отдай Claude Design только визуал
5. **Итеративно правь.** Если модель сделала не так — не переделывай с нуля, пиши в чат правку. Контекст прототипа держится.
6. **Примеры лучше описаний.** «Заголовок как в Zara Home» работает хуже, чем прикрепить скриншот Zara Home.

## 7. Ограничения (грабли)

- ❌ **Нет экспорта видео в файл** — только screen recording
- ❌ **Нет обратного экспорта в Figma** — только импорт через .fig
- ❌ **НЕ генерирует картинки** — загружаешь сам или через Claude Code
- ❌ **Сложные React-приложения с БД не импортирует** — только одностраничники
- ❌ **Интерфейс только на английском** (промпты и комментарии — на русском работают)
- ⚠️ **Accessibility не из коробки** — сгенерированный код не соответствует WCAG/508 (блокер для гос/медицины/крупных корп)
- ⚠️ **Генерируемый код захардкоженный** (inline CSS, не компонентная структура) — сложно дорабатывать
- ⚠️ **Слайды статичны без явных указаний** — надо просить анимации отдельно
- ⚠️ **Без Design System — однотипные «усреднённые» результаты** (12 пользователей это отметили независимо)
- ⚠️ **Импорт .fig формально работает, но может отказывать** на отдельных файлах
- ⚠️ **Bold в Gothra нет** — модель может синтезировать, надо явно запрещать
- ⚠️ **Ручное редактирование кода ограничено** — только цветовая палитра и субтитры
- ⚠️ **PDF > 10-15 МБ может не пройти** в Project Knowledge
- ⚠️ **Русский в именах файлов** иногда капризничает — лучше латиница

**Практические выводы:**
- Pro тариф → только для редких простых задач
- Max 5x+ → можно работать, но следить за сценарием
- Edit-режим (4.5) + Tweaks на готовое → экономит лимит в разы

### 7.5.1 Что Max 5x ($100) значит конкретно для YHATA

**Расчёт бюджета (оценка, не официальный):**
- 1 полный прогон на Pro = ~65% недельного лимита
- Max 5x = ×5 больше Pro → 1 полный прогон = ~13% недельного
- Примерно **7-8 полных прогонов в неделю** (Design System + Prototype) — но только при дорогих действиях
- На простых итерациях (Comment-правки на готовом) — сильно больше

**Красные линии (когда упрёмся):**
- Несколько пересозданий Design System с нуля в день → съест быстро
- Большие prototypes (5+ экранов) в High Fidelity → дорого
- Частые итерации через Comment вместо Edit → утекает незаметно

**Рекомендации работы на Max 5x:**
1. **Design System делаем один раз и правильно** — не переделывать, а дорабатывать через UX-проверку компонентов
2. **Промпты для концепций, Edit для мелочей** — правило жёсткое
3. **Tweaks-ползунки** вместо описаний стиля «сделай минималистичнее» — быстрее и экономнее
4. **Handoff в Claude Code после первого успешного макета** — дальше дорабатываем в коде, не в Claude Design (экономит лимит Design, тратим лимит Code, который отдельный)
5. **В день запуска нового прототипа** — не запускать ещё что-то тяжёлое параллельно
6. **Отслеживать индикатор лимита** в интерфейсе. Если выше 70% — до понедельника идти в Edit-режим

## 8. Позиционирование vs конкурентов

### Что говорят официально и в сообществе

| Позиция | Источник | Валидация |
|---|---|---|
| «Убийца Figma» | Маркетологи-блогеры | ❌ преувеличение |
| «Замена Canva» | Дизайнеры в комментариях | ⚠️ частично правда |
| «Дополнение, не замена» | Официальная позиция Anthropic | ✓ но хитрая — см. ниже |
| «Это Claude Code с обёрткой» | Критический обзор | ❌ упрощение (разные архитектуры) |

**Факты (а не мнения):**
- Акции Figma упали **7%** в день релиза (17.04.2026) — подтверждено VentureBeat/Gizmodo
- CTO Anthropic Майк Кригер вышел из совета директоров Figma **за 3 дня до релиза** (VentureBeat) — косвенный сигнал
- С Canva — **двухлетнее стратегическое партнёрство** (пресс-релиз Canva через BusinessWire). Canva Foundation Design Model обрабатывает экспортный поток. Canva MCP для Claude — с июля 2025

**Самая взвешенная формулировка:** Claude Design — **другой класс инструмента**, не прямая замена Figma. Для профессионального дизайна — нет. Для быстрых прототипов и «на вчера» — конкурирует с Canva, возможно превосходит.

### Таблица сравнения с конкурентами

| Инструмент | Движок | Фокус | vs Claude Design |
|---|---|---|---|
| **Figma Make** (с мая 2025) | **Claude 3.7 Sonnet** (!) | Внутри Figma, React+Tailwind | Парадокс: конкурент Anthropic тоже на Anthropic-модели |
| **Lovable** | Свой | Полные web-приложения с БД/авторизацией/платежами | У Claude Design этого нет |
| **V0 Vercel** | Свой | UI-генерация для разработчиков, деплой в Vercel | Claude Design — для дизайнеров, не разработчиков |
| **Canva Magic Studio** | Свой + Claude | Графика, соцсети, шаблоны | Claude Design теперь интегрирован с Canva |
| **Google Stitch** | Google | Дизайн-прототипы | Конкурент, но уже |
| **Replit** | Разные | Полный dev-цикл | Не для дизайна |

**Главное отличие Claude Design** по консенсусу обзоров: работа **в твоей дизайн-системе**, а не выдача усреднённого шаблона.

## 8.5 Когда Claude Design подходит, а когда нет

### Подходит:
- ✓ Не-дизайнер + нужен приличный результат быстро
- ✓ Брендированный лендинг (с настроенной Design System)
- ✓ Быстрая презентация с текстом речи (Speaker Notes)
- ✓ Разработчик + нужен UI-прототип → Claude Code handoff
- ✓ Одноразовые материалы (постеры, меню, флаеры, визитки)
- ✓ Max/Team тариф без проблем с лимитами

### НЕ подходит:
- ✗ Профессиональному дизайнеру Figma (нет ручного контроля, шаблонность)
- ✗ Сложный React с базой данных (не импортирует)
- ✗ Accessibility требуется (WCAG/508 не из коробки)
- ✗ Pro-тариф + регулярная работа каждый день (лимиты)
- ✗ Нужен прямой экспорт в Figma (его нет)
- ✗ Ожидаемый уровень — дизайн-агентство (качество среднее)
- ✗ Аудитория — английский не знают (UI только на EN)

## 11. Ключевые правила YHATA для промптов (шпаргалка)

**NEVER:**
- Шрифты кроме Gothra + Georgia fallback
- Bold (его нет в Gothra)
- Emoji в UI
- Градиенты, glassmorphism, 3D blobs, mesh backgrounds
- Яркие/неоновые цвета
- Восклицательные знаки в копирайте
- Жирные CTA («КУПИТЬ СЕЙЧАС!!!»)
- Shadow opacity > 15%
- Filled/цветные иконки (только line, monochrome, Lucide)

**ALWAYS:**
- Палитра Quiet Order (брендовое) или App (продуктовое)
- Gothra Text VF, slnt axis для italic
- Soft shadows 6-10% opacity
- Воздух: минимум 24-32px padding в карточках
- Touch target ≥ 44px
- Mobile-first, 360px min
- Safe area учтена

## 14. FAQ (частые вопросы из сообщества)

**Можно ли писать промпты на русском?** Да. Также UX-оценки компонентов Design System можно комментировать по-русски — Claude понимает. Интерфейс (меню) — только английский.

**Есть ли мобильная/десктопная версия?** Нет. Только веб-браузер, claude.ai/design. В десктопной программе Claude появляется иконка Claude Design, но она открывает тот же веб-интерфейс.

**Прямой экспорт в Figma есть?** Нет. Только импорт .fig. Дизайнеры, желающие «черновик в Claude Design → доводка в Figma» — столкнутся с проблемой.

**Расходуется ли токен на каждое действие?** Нет. Режим **Edit** (прямая правка свойств) не задействует модель. Comment/Draw/Tweaks — задействуют.

**Можно ли делать мобильные приложения?** Прототипы — да, готовый код работающего приложения — нет. Нужен отдельный этап разработки (через Claude Code handoff).

**Несколько брендов одновременно?** В командных аккаунтах можно держать несколько Design Systems и переключаться между ними. Удобно агентствам.

**Время создания Design System?** Сам интерфейс показывает «~15 минут». Реально — от 5 до 15 мин в зависимости от объёма загруженных материалов.
